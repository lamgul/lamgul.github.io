논문 분석: ResNet — 깊이의 역설을 지름길로 풀다

2021년 6월• ResNet · CNN · 논문

ETRI에서 객체탐지를 처음 배울 때, 모델 설정 파일의 backbone 칸에는 늘 resnet이 적혀 있었습니다. 왜 다들 ResNet인가 — 그 답은 2015년의 역설 하나로 거슬러 올라갑니다. 층을 더 쌓았더니 성능이 훈련 오차부터 나빠지는 현상, 그리고 그것을 덧셈 한 줄로 풀어낸 논문. He 등의 원 논문을 다시 읽고, 핵심 주장을 토이 계산으로 하나씩 확인한 기록입니다.

출처

Deep Residual Learning for Image Recognition

Kaiming He, Xiangyu Zhang, Shaoqing Ren, Jian Sun · CVPR 2016 · arXiv:1512.03385

PDF ↗ · arXiv ↗

먼저 짚고 가기
퇴화 문제Degradation
층을 더 쌓았는데 시험 성적은 물론 훈련 성적부터 나빠지는 현상입니다. 훈련 데이터를 더 못 외우게 되는 것이라, 너무 잘 외워서 탈이 나는 과적합과는 다른 병입니다. 이 글 전체가 이 병의 진단과 처방입니다.
항등 사상Identity Mapping
입력을 아무것도 바꾸지 않고 그대로 내보내는 함수 y = x입니다. "아무것도 하지 않기"인 셈인데, 신경망에게는 이것이 의외로 어려운 목표라는 것이 이 이야기의 반전입니다.
잔차Residual
목표값에서 이미 가진 값을 뺀 나머지, 곧 "더 고쳐야 할 양"입니다. ResNet은 층에게 완성본을 통째로 만들게 하는 대신 이 수정량만 배우게 합니다. 이름의 R이 여기서 옵니다.
지름길 연결Shortcut Connection
층 몇 개를 건너뛰어 입력을 출력에 그대로 더해 주는 연결선입니다. 학습할 파라미터가 하나도 없어서, 드는 비용이 덧셈뿐입니다.
역전파Backpropagation
출력에서 생긴 오차를 입력 방향으로 거꾸로 흘려보내며 층마다 수정량(기울기)을 계산하는 절차입니다. 층을 지날 때마다 국소 기울기가 곱해지는 구조라, 이 곱셈이 글 후반의 주인공입니다.

1. 56층이 20층에게 진다

2012년 AlexNet(8층)이 판을 뒤집은 뒤, 2014년의 VGG는 19층, GoogLeNet은 22층으로 깊이를 늘리며 정확도를 끌어올렸습니다. "깊이가 곧 표현력"이라는 통념이 자리 잡던 때죠. 그렇다면 층을 계속 쌓으면 계속 좋아질까요? 논문 1장은 정확히 이 질문으로 시작하고, 곧바로 찬물을 끼얹는 실험을 보여줍니다. CIFAR-10에서 지름길 없이 평범하게 쌓은(plain) 20층과 56층을 비교하면 — 56층이 훈련 오차와 테스트 오차 모두에서 더 나쁩니다(1장 그림 1).

테스트만 나빠졌다면 익숙한 과적합이었을 겁니다. 하지만 훈련 오차부터 높다는 건, 더 큰 모델이 훈련 데이터조차 더 못 외운다는 뜻입니다. 용량은 남아도는데도요. 논문은 이 현상에 퇴화(degradation)라는 이름을 붙입니다. 그리고 흔한 용의자인 기울기 소실 탓으로 돌리기도 어렵다고 짚습니다 — 정규화된 초기화와 배치 정규화 덕분에 이 정도 깊이는 수렴 자체는 되는 상태에서 벌어지는 일이기 때문입니다(1장). 수렴은 하는데, 나쁜 곳으로 수렴한다. 이것이 2015년의 미스터리였습니다.

2. 반증은 논리 안에 있다 — 항등 사상 사고실험

제게 이 논문에서 가장 아름다운 대목은 실험이 아니라 1장의 사고실험입니다. 잘 학습된 20층 모델이 있다고 합시다. 56층 모델을 만들 때 앞 20층에 그 모델을 그대로 복사해 넣고, 나머지 36층을 항등 사상 — 입력을 그대로 통과시키는 층 — 으로 채우면 어떻게 될까요? 이 깊은 모델은 최소한 얕은 모델과 같은 훈련 오차를 냅니다. 즉 깊은 모델이 더 나쁠 이유는 논리적으로 존재하지 않습니다. 더 나은(적어도 같은) 해가 구성적으로 존재하는데, 현재의 최적화기가 그 해를 찾아내지 못할 뿐입니다.

이로써 범인의 윤곽이 잡힙니다. 표현력의 문제가 아니라 최적화의 문제이고, 그 한가운데에 "비선형 층 여러 개로 항등 사상을 흉내 내기"의 어려움이 있습니다. ReLU(Wx) 꼴의 층에게 항등은 저절로 되는 일이 아니라, 특별한 가중치 조합을 정확히 맞혀야 하는 과녁입니다. "아무것도 하지 않기"가 신경망에게는 가장 어려운 주문 중 하나인 셈입니다.

3. H(x) 대신 F(x) = H(x) − x를 배운다

ResNet의 답은 모델을 키우는 게 아니라 목표를 바꾸는 것입니다. 층 더미가 배우길 바라는 이상적인 사상을 H(x)라 합시다. 논문 3.1절은 층에게 H를 직접 맞히게 하는 대신 잔차 F(x) = H(x) − x를 맞히게 하자고 제안합니다. 원래 목표는 지름길로 건너온 입력을 더해서 복원합니다 — y = F(x) + x(3.2절 식 1). 항등이 최적인 자리라면 이제 할 일은 F를 0으로 미는 것뿐입니다. 가중치를 0 쪽으로 누르는 건 학습에게 가장 자연스러운 방향이죠. "아무것도 하지 않기"가 어려운 목표에서 기본값으로 바뀝니다. 입력과 출력의 차원이 달라지는 자리에서만 지름길에 투영 행렬을 끼우고(식 2), 그 외의 지름길에는 추가 파라미터도 추가 연산도 없습니다(3.2절) — 덧셈 하나뿐.

이 차이가 실제로 무엇을 만드는지, 학습 초기 상태(작은 가중치)의 블록 30개를 쌓아 신호를 통과시켜 봅니다.

import numpy as np

def relu(v): return np.maximum(0.0, v)

def plain_block(x, W1, W2):             # H(x)를 통째로 배워야 한다
    return relu(W2 @ relu(W1 @ x))

def residual_block(x, W1, W2):          # H(x) = F(x) + x — 논문 식 (1)
    return W2 @ relu(W1 @ x) + x         # 덧셈 뒤 ReLU는 크기 비교를 위해 생략

rng = np.random.default_rng(42)
x = rng.normal(size=64)                   # ‖x‖ ≈ 6.32
h = r = x
for _ in range(30):                       # 블록 30개를 쌓는다
    W1 = rng.normal(scale=0.01, size=(64, 64))   # 학습 초기의 작은 가중치
    W2 = rng.normal(scale=0.01, size=(64, 64))
    h = plain_block(h, W1, W2)
    r = residual_block(r, W1, W2)

print(np.linalg.norm(h))                  # 4.6e-75 — 신호가 흔적도 없다
print(np.linalg.norm(r))                  # 6.33 — 입력이 거의 그대로 살아 있다

plain 더미에서는 서른 블록 만에 신호가 10−75 수준으로 증발합니다. 1보다 훨씬 작은 변환이 서른 번 곱해졌기 때문입니다. residual 더미에서는 각 블록의 F가 거의 0이니 x가 그대로 살아서 끝까지 갑니다. "항등이 기본값"이라는 말의 물리적 의미가 이 두 줄의 norm 차이입니다. 참고로 50층 이상의 ResNet은 3×3 두 장 대신 1×1–3×3–1×1의 병목 블록으로 같은 아이디어를 더 싼 연산으로 굴립니다(4.1절).

*

지름길의 파라미터가 0개라는 사실은 수사가 아니라 전략입니다. 파라미터 수와 연산량이 같으니 plain망과의 비교가 공정해졌고(3.2절), 어떤 기존 구조에든 비용 없이 끼워 넣을 수 있었습니다. 좋은 설계 장치는 성능을 말하기 전에 도입 장벽부터 없앱니다.

4. 기울기에게 난 지름길 — 곱셈의 세계에 덧셈 하나

순전파에서 신호를 살리는 덧셈은, 역전파에서는 기울기의 지름길이 됩니다. y = F(x) + x를 x로 미분하면 자코비안은 ∂F/∂x + I — 학습된 가지가 무슨 값을 갖든 항등 행렬 I가 항상 더해져 있습니다. 깊은 망의 역전파는 층별 국소 기울기의 곱이라서, plain망에서는 크기가 1보다 작은 인자가 56번 곱해지면 기울기가 지수적으로 쪼그라듭니다. residual망에서는 곱의 인자가 (1 + 작은 것)이라 1 근처에 머뭅니다. 숫자로 보면 이렇습니다.

L = 56                                   # 층 수
a = 0.90                                 # plain 층 하나의 국소 기울기 크기 (|a| < 1)
print(a ** L)                            # 0.0027 — 출력 기울기의 0.3%만 1층에 도달

# residual: 층의 자코비안이 (1 + 가지의 기울기 f). f는 작고 부호가 섞인다
f = 0.10 * np.where(np.arange(L) % 2 == 0, 1.0, -1.0)
print(np.prod(1 + f))                    # 0.75 — 1 근처에 머문다

# 곱을 전개하면: (1+f₁)(1+f₂)··· = 1 + Σfᵢ + ···
# 맨 앞의 "1"이 어떤 곱셈도 거치지 않고 직행하는 항등 경로다

전개식의 맨 앞에 있는 1이 지름길의 정체입니다. 출력의 기울기가 어떤 층의 가중치와도 곱해지지 않은 채 1층까지 직행하는 경로가 항상 하나 확보되어 있는 것이죠. 아래 위젯에서 깊이를 직접 올려 보세요. plain의 막대는 깊이 56에서 눈으로 볼 수 없을 만큼 사라지지만, residual로 전환하면 같은 깊이에서도 막대가 유지됩니다.

1층(입력 쪽)에 도달하는 기울기: —

막대 = 그 층에 도달하는 역전파 기울기 크기(출력 = 1). 결정적 토이 계산 — plain은 |국소 기울기| < 1의 곱셈 감쇠, residual은 (1 + 작은 가지 기울기)의 곱입니다.

*

정직하게 덧붙이면 — 논문 4.1절은 배치 정규화를 쓴 plain망의 역전파 신호가 건강한 크기임을 확인하고, 퇴화의 원인이 기울기 소실일 가능성은 낮다고 스스로 선을 긋습니다. 저자들의 공식 진단은 "깊은 plain망은 지수적으로 낮은 수렴 속도를 가질지 모른다"는 추측에서 멈춥니다. 위 토이 계산과 위젯은 소실의 재연이라기보다, 곱셈만 있는 구조와 덧셈 경로가 있는 구조의 차이가 최적화 난이도를 어떻게 가르는지에 대한 직관으로 읽어 주세요.

5. 숫자는 어떻게 답했나 — 표 2와 3.57%

설계가 아무리 우아해도 답은 숫자가 합니다. ImageNet에서 plain망 18층과 34층을 비교하면 34층의 top-1 오차가 오히려 높습니다 — 27.94% 대 28.54%(4.1절 표 2, 10-crop 검증). 퇴화가 대규모 데이터에서도 재현된 것이죠. 같은 구조에 지름길만 더한 ResNet은 이 순서를 뒤집습니다. 18층 27.88%, 34층 25.03% — 깊이가 다시 자산이 됐고, plain-34 대비 top-1 오차가 3.5%p 낮아졌습니다(4.1절).

깊이의 천장도 함께 뚫립니다. ResNet-152는 VGG망보다 8배 깊지만 연산 복잡도는 오히려 낮고(초록), 앙상블은 ImageNet 테스트에서 top-5 오차 3.57%로 ILSVRC 2015 분류 부문 1위를 가져갑니다(초록). 그런데 제 눈에 가장 인상적인 숫자는 따로 있습니다. 검출기 구조는 그대로 두고 backbone만 ResNet으로 바꿨더니 COCO 객체탐지에서 28% 상대 개선(초록). ETRI에서 만졌던 설정 파일의 backbone: resnet50 한 줄이 하는 일이 정확히 이것입니다. 탐지라는 머리가 아무리 좋아도 특징이라는 몸통이 얕으면 소용없다는 것 — 데이터 가공을 하며 어렴풋이 느끼던 감각에, 논문이 숫자를 붙여 줬습니다.

CIFAR-10 실험(4.2절)에는 흥미로운 각주가 하나 있습니다. 110층 ResNet은 6.43% 오차로 잘 학습되지만, 1202층은 수렴은 하는데 110층보다 나쁜 7.93%에 그칩니다. 저자들은 이번엔 과적합을 지목합니다. 최적화라는 병목이 풀리자 그다음 병목이 모습을 드러낸 것입니다. 문제는 사라지지 않고 자리를 옮깁니다.

6. 정리 — 덧셈 하나가 바꾼 것

  • 진단이 해법보다 먼저다. 퇴화가 과적합도, (정규화 이후의) 기울기 소실도 아니라 최적화의 문제임을 사고실험으로 좁혔기에, 해법이 "항등을 기본값으로"라는 한 점을 정확히 겨냥할 수 있었다.
  • 배움의 대상을 바꾸는 것이 설계다. H(x)를 통째로 배우게 하는 대신 "x에서 얼마나 고칠까"(잔차)만 배우게 했다. 모델을 키우는 대신 문제를 쉽게 만드는 것 — 좋은 구조의 정의라고 생각합니다.
  • 비용 없는 장치는 어디에나 스며든다. 파라미터 0개짜리 덧셈이라 모든 아키텍처가 부담 없이 가져다 썼다. 이후 Transformer 블록마다 들어 있는 Add에도 같은 지름길이 있다.

ETRI에서 backbone은 제게 그냥 고유명사였습니다. 이 논문을 읽고 나서야 그 이름이 왜 모든 설정 파일에 적혀 있는지 — 깊이가 왜 2015년 이전에는 부채였고 이후에는 자산이 됐는지 — 설명할 수 있게 됐습니다. 덧셈 하나가 바꾼 세계입니다.
