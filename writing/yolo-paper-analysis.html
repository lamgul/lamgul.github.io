논문 분석: YOLO — 검출을 회귀 문제로 바꾼 한 발상

2023년 1월• YOLO · Object Detection · 논문

2023년, 사전학습된 YOLOv5로 2주 만에 주차장 차량감지 MVP를 만들었습니다. 모델은 잘 돌아갔지만, 그때의 저는 왜 "한 번만 보는" 게 가능한지를 설명하지 못했습니다. 그래서 원점으로 돌아갔습니다 — Redmon 등의 원 논문 "You Only Look Once"(2015)를 다시 읽고, 격자 분할·바운딩박스 회귀·NMS를 numpy로 직접 만들어 본 기록입니다.

출처

You Only Look Once: Unified, Real-Time Object Detection

Joseph Redmon 외 · CVPR 2016 · arXiv:1506.02640

공식 PDF ↗ · arXiv ↗

먼저 짚고 가기
객체 검출Object Detection
사진 속에 무엇이 있는지(분류)에 더해 어디에 있는지(위치)까지 상자로 찾아내는 일입니다. 이 글은 그 "찾는 방식"을 통째로 바꾼 논문 이야기입니다.
바운딩박스Bounding Box
물체를 딱 감싸는 네모 상자로, 보통 중심 좌표와 너비·높이로 표현합니다. YOLO가 최종적으로 내놓는 결과물이 바로 이 상자들입니다.
IOUIntersection over Union
두 상자가 겹친 넓이를 둘을 합친 넓이로 나눈 값(0~1)입니다. 1에 가까울수록 두 상자가 잘 포개진 것이라, "예측이 정답을 얼마나 잘 맞췄나"를 재는 자로 쓰입니다.
NMSNon-Maximum Suppression, 비최대 억제
한 물체에 상자 여러 개가 겹쳐 나올 때, 점수가 가장 높은 하나만 남기고 많이 겹치는 나머지를 지우는 후처리입니다. 중복 상자를 걷어내 결과를 깔끔하게 만듭니다.
mAPmean Average Precision
검출 정확도를 하나의 숫자로 요약한 지표로, 높을수록 잘 맞힌 것입니다. 본문에서 "63.4 mAP"처럼 모델끼리 정확도를 비교하는 잣대로 등장합니다.

1. 2015년의 검출은 두 단계였다

당시 객체 검출의 주류는 분류기를 재활용하는 구조였습니다. R-CNN 계열은 먼저 Selective Search로 약 2,000개의 후보 영역을 뽑고, 각 영역을 CNN 분류기에 통과시킨 뒤, 박스를 정제하고 중복을 지우는 후처리를 거칩니다. 후보 생성 → 분류 → 후처리가 각각 따로 학습·튜닝되는 복잡한 파이프라인이고, 무엇보다 느렸습니다. 논문 기준으로 Fast R-CNN은 70.0 mAP를 내지만 초당 0.5장 — 이미지 한 장에 2초입니다.

YOLO의 답은 파이프라인을 통째로 접는 것이었습니다. 논문 표현 그대로, 검출을 "이미지 픽셀에서 바운딩박스 좌표와 클래스 확률로 가는 단일 회귀 문제"로 다시 정의합니다. 네트워크를 이미지에 한 번 통과시키면 검출이 끝납니다. 그 결과가 Titan X 기준 base 모델 45fps에 VOC 2007 63.4 mAP, 축소판 Fast YOLO는 155fps에 52.7 mAP입니다. 같은 표에서 Faster R-CNN(VGG-16)은 73.2 mAP로 더 정확했지만 7fps였습니다. 정확도를 조금 내주고 실시간이라는 판을 새로 연 것입니다.

2. 한 번만 본다 — 7×7 격자와 텐서 하나

핵심 설계는 단순합니다. 이미지를 S×S 격자(논문은 S=7)로 나누고, 객체의 중심이 떨어진 셀이 그 객체의 검출을 책임집니다. 각 셀은 바운딩박스 B개(논문은 B=2)를 예측하는데, 상자 하나당 x, y, w, h, confidence 다섯 값입니다. x, y는 셀 기준 오프셋(0~1), w, h는 이미지 전체 기준 비율이고, confidence는 Pr(Object)·IOU — "여기 뭔가 있고, 내 상자가 그걸 얼마나 잘 감쌌나"입니다. 클래스 확률 C개(VOC는 20)는 상자가 아니라 셀마다 한 벌만 예측합니다. 그래서 출력은 7×7×(2·5+20) = 7×7×30 텐서 하나입니다.

네트워크는 GoogLeNet에서 영감을 받은 합성곱 24층 + 완전연결 2층이고, 검출 시 입력은 448×448입니다. 구조 그림은 화려하지만 요지는 하나입니다 — 이미지가 들어가면 7×7×30이 나온다. 학습이 되려면 정답도 같은 모양이어야 하니, 정답 인코딩부터 만들어 봅니다.

import numpy as np

S, C = 7, 20                                  # 격자 7×7, 클래스 20개 (VOC)

def encode_target(boxes, labels, S=S, C=C):
    # boxes: (N, 4) — [cx, cy, w, h], 이미지 기준 0~1 정규화
    target = np.zeros((S, S, 5 + C))
    for (cx, cy, w, h), cls in zip(boxes, labels):
        col, row = int(cx * S), int(cy * S)       # 중심이 속한 셀
        x, y = cx * S - col, cy * S - row         # 셀 내부 오프셋 0~1
        target[row, col, :5] = [x, y, w, h, 1.0]  # 다섯 번째 = 객체 존재
        target[row, col, 5 + cls] = 1.0           # 클래스 one-hot
    return target

car = np.array([[0.52, 0.48, 0.30, 0.42]])      # 화면 중앙의 차 한 대
t = encode_target(car, [6])                       # VOC에서 6 = car
print(t.shape)                                    # (7, 7, 25) — 정답은 셀당 상자 1개
print(np.argwhere(t[..., 4] == 1))               # [[3 3]] — (3,3) 셀이 책임진다

이 인코딩을 손으로 짜 보면 "회귀 문제로 바꿨다"는 말이 구체적으로 잡힙니다. 검출이라는 문제가 고정 크기 텐서를 맞히는 문제로 바뀐 것이고, 고정 크기 출력은 곧 마지막 완전연결층에서 한 번에 뽑을 수 있다는 뜻입니다. 후보 영역도, 단계도 없습니다.

3. 손실 함수의 디테일 — √w가 있는 이유

손실은 전 항목 제곱합 오차입니다. 최적화가 쉽다는 이유인데, 논문은 그대로 쓰면 생기는 문제를 셋 짚고 각각 장치를 답니다. 첫째, 49개 셀 대부분에는 객체가 없어서 confidence를 0으로 미는 기울기가 압도합니다. 그래서 좌표 오차에 λ_coord=5를 곱해 키우고, 객체 없는 셀의 confidence 오차에는 λ_noobj=0.5를 곱해 줄입니다. 둘째, 셀의 상자 2개 중 정답과 IOU가 가장 높은 쪽 하나만 "responsible"로 지정해 좌표 손실을 받게 합니다 — 학습이 진행되면 두 예측기가 서로 다른 크기·비율에 특화됩니다.

셋째가 제일 흥미로운 부분입니다. 큰 상자의 5px 오차와 작은 상자의 5px 오차는 같은 손실이지만 같은 잘못이 아닙니다. 논문의 처방은 w, h 대신 √w, √h를 회귀하는 것입니다. 숫자로 확인해 봅니다.

big, small = 0.60, 0.06            # 너비 60%짜리 상자, 6%짜리 상자
err = 0.03                          # 둘 다 3%p씩 예측이 빗나갔다고 하자

# w를 그대로 회귀하면 — 손실이 똑같다
print(err ** 2, err ** 2)                            # 0.0009  0.0009

# √w를 회귀하면 — 작은 상자의 실수가 8배쯤 비싸진다
print((np.sqrt(big + err) - np.sqrt(big)) ** 2)      # ≈ 0.00037
print((np.sqrt(small + err) - np.sqrt(small)) ** 2)  # ≈ 0.00303

제곱근 하나로 작은 상자의 같은 오차가 8배 넘게 비싸집니다. 완전한 해결은 아니고 — 뒤에서 보듯 작은 객체의 위치 오차는 끝까지 YOLO의 주된 약점입니다 — 방향을 맞춘 절충입니다. Attention 논문의 √d_k처럼, 수식의 자잘한 상수와 변형에는 다 이유가 있다는 걸 여기서도 확인했습니다.

4. 마지막 관문 — 텐서를 상자로, 그리고 NMS

추론 시에는 출력 텐서를 상자 목록으로 되돌립니다. 셀마다 상자 2개씩 7×7×2 = 98개 후보가 나오고, 각 상자의 점수는 클래스 확률과 confidence의 곱 — Pr(Class_i)·IOU에 해당하는 값입니다.

def decode(pred, S=7, B=2, C=20, score_thr=0.2):
    boxes, scores, classes = [], [], []
    for row in range(S):
        for col in range(S):
            cell = pred[row, col]                    # 길이 30짜리 벡터
            cls_prob = cell[B * 5:]                  # Pr(Class|Object) — 셀당 한 벌
            for b in range(B):
                x, y, w, h, conf = cell[b * 5:(b + 1) * 5]
                score = conf * cls_prob.max()        # Pr(Object)·IOU × Pr(Class|Object)
                if score < score_thr:
                    continue
                cx, cy = (col + x) / S, (row + y) / S  # 셀 좌표 → 이미지 좌표
                boxes.append([cx - w/2, cy - h/2, cx + w/2, cy + h/2])
                scores.append(score)
                classes.append(cls_prob.argmax())
    return np.array(boxes), np.array(scores), np.array(classes)

큰 객체는 여러 셀에 걸치기 마련이라 후보들이 서로 겹칩니다. 이 중복을 지우는 게 비최대 억제(NMS)입니다. 점수가 가장 높은 상자를 살리고, 그것과 많이 겹치는(IOU가 임계값 이상인) 나머지를 지우는 일을 반복합니다. 논문도 NMS로 mAP가 2~3%p 오른다고 언급합니다.

def iou(a, b):
    # a, b: [x1, y1, x2, y2]
    ix1, iy1 = max(a[0], b[0]), max(a[1], b[1])
    ix2, iy2 = min(a[2], b[2]), min(a[3], b[3])
    inter = max(0.0, ix2 - ix1) * max(0.0, iy2 - iy1)
    area = lambda r: (r[2] - r[0]) * (r[3] - r[1])
    return inter / (area(a) + area(b) - inter + 1e-9)

def nms(boxes, scores, iou_thr=0.5):
    order = list(np.argsort(scores)[::-1])        # 점수 내림차순
    keep = []
    while order:
        best = order.pop(0)                       # 가장 확신하는 상자는 살린다
        keep.append(best)
        order = [i for i in order
                 if iou(boxes[best], boxes[i]) < iou_thr]  # 많이 겹치면 제거
    return keep

pred = np.random.rand(7, 7, 30)                  # 네트워크 출력이라 치자
b, s, c = decode(pred)
print(len(s), len(nms(b, s)))                     # 문턱 넘은 후보 수 → NMS 뒤 남은 수
*

차량감지 MVP에서 오검출·미검출을 조정할 때 처음 손댄 것은 모델이 아니라 위 코드의 score_thr와 iou_thr 두 개였습니다. 임계값을 낮추면 놓침이 줄고 오검출이 늘어납니다 — 그 트레이드오프가 어느 줄에서 생기는지 알고 만지는 것과 모르고 만지는 것은 다릅니다. 검출 라이브러리의 conf·iou 파라미터가 하는 일이 정확히 이것입니다.

NMS 시뮬레이터 — 임계값 두 개가 상자를 거르는 법직접 조작해 보세요

후보 8개 → 신뢰도 통과 8 · NMS 억제 3 · 생존 5

생존 IoU로 억제 신뢰도 미달

위 nms() 코드와 똑같이 매번 실제 IoU를 계산합니다 — IoU를 0.20까지 내리면 차 한 대에 상자 하나로 정리되지만, 겹치는 상대가 없는 배경 오검출(0.28)은 신뢰도를 0.30까지 올려야 사라집니다

5. 공짜는 없다 — 격자가 지불한 대가

논문은 한계 절에서 대가를 스스로 밝힙니다. 셀 하나는 상자 2개, 클래스는 한 벌만 예측하므로, 한 셀 안에 중심이 여러 개 떨어지면 물리적으로 다 잡을 수 없습니다. 논문이 드는 예가 새 떼처럼 무리 지은 작은 객체입니다. 여러 번의 다운샘플링을 지나며 거칠어진 특징으로 상자를 그리는 데다, √w로 완화했어도 작은 상자의 위치 오차가 주된 오류원으로 남습니다. 대신 얻은 것도 명확합니다. 후보 영역만 보는 방식과 달리 이미지 전체를 보고 추론하므로 문맥이 반영되고, 배경을 객체로 오인하는 오류가 Fast R-CNN의 절반 이하였습니다.

이 한계 목록은 제게 이론이 아니라 체크리스트였습니다. 주차장 CCTV 화면 속 차량은 크고, 주차 구역 덕에 간격이 일정해서 한 셀에 두 대가 몰릴 일이 거의 없습니다 — YOLO의 약점 목록에 하나도 걸리지 않는 문제였던 겁니다. 사전학습 모델을 그대로 쓰는 결정이 편했던 이유가 논문에 이미 적혀 있었습니다. 제가 쓴 YOLOv5는 여러 해상도의 격자와 앵커로 v1의 한계를 상당 부분 완화했지만, "격자가 담을 수 있는 밀도까지만 잡는다"는 트레이드오프의 뼈대는 지금도 같습니다.

*

검출 모델을 고르기 전에, 여러분의 데이터에서 객체 중심을 격자에 뿌려 한 셀에 2개 이상 몰리는 비율부터 재 보세요. 2절의 encode_target에서 덮어쓰기가 발생하는 비율을 세면 되니 다섯 줄이면 됩니다. 그 비율이 높다면 답은 하이퍼파라미터가 아니라 입력 해상도·타일링·더 촘촘한 격자의 모델입니다.

6. 논문에서 실무로 — 2주짜리 MVP의 뒷배

대흥정보에서의 차량감지 MVP는 이 논문 계보의 수혜였습니다. 사전학습된 '차량' 클래스를 필터링하고 바운딩박스 좌표로 주차 구역 점유를 판단하는 구조는, 단일 네트워크가 상자·점수·클래스를 한 번에 내주기에 가능한 설계였고, 실시간 30fps라는 요구도 "You Only Look Once"라는 제목이 곧 답이었습니다. 돌아보면 그보다 앞서 ETRI에서 Vision AI 분야 연구에 참여했을 때도 검출은 늘 파이프라인의 출발점이었습니다. 그때는 도구로만 썼던 그 출발점을, 이제는 이 논문의 격자 그림으로 설명할 수 있습니다.

  • 문제를 다시 정의하는 것이 기여다. YOLO는 더 좋은 분류기를 만든 게 아니라, 검출을 텐서 하나를 맞히는 회귀로 바꿨다. 프레이밍이 파이프라인을 접었다.
  • 한계 절이 실무의 체크리스트가 된다. 작은 객체·군집·격자 밀도 — 내 데이터가 그 한계에 걸리는지부터 보면, 모델 선택이 유행이 아니라 판단이 된다.
  • 미니 구현 수십 줄의 값어치. 인코딩·디코딩·NMS를 직접 짜 보면 라이브러리의 conf·iou 파라미터가 어느 수식에서 왔는지 읽힌다. 그다음부터 튜닝은 감이 아니다.
