Feature Engineering 실전 — 불쾌지수가 강수량을 이긴 이유

2024년 6월• EDA · 다중공선성 · 파생변수

전력 예측 대회에 나갔을 때, 같은 데이터와 같은 모델을 놓고 순위를 가른 것은 데이터를 어떻게 이해했는가였습니다. 저는 처음에 기상 데이터를 다 넣으면 되겠거니 생각하고 시작했는데, EDA를 해 보니 변수들이 서로 겹쳐 있었습니다. 그때 기온과 습도를 불쾌지수 하나로 묶으며 정리했던 과정을 적어 둡니다.

먼저 짚고 가기
EDAExploratory Data Analysis
모델을 적합하기 전에 요약통계와 그림으로 데이터의 분포·결측·이상치·변수 간 관계를 먼저 확인하는 분석 단계입니다. Tukey가 1977년 같은 제목의 책에서, 가설을 미리 세우고 검정하는 확증적 분석(confirmatory data analysis)과 짝을 이루는 절차로 정리했습니다. 여기서 나오는 것은 결론이 아니라 가설이라, 눈에 띈 패턴을 그대로 결론으로 삼으면 표본에서 우연히 생긴 모양을 사실로 착각하게 됩니다. 이 글에서 상관 행렬을 뽑아 본 것이 이 단계에 해당합니다.
상관계수correlation, r
흔히 쓰는 피어슨 상관계수 r은 두 변수의 공분산을 각자의 표준편차 곱으로 나눈 값으로, 항상 −1에서 1 사이에 놓입니다. 재는 것은 두 변수의 선형 관계의 방향과 강도뿐이라, U자처럼 휜 관계는 실제로 강해도 r이 0 근처로 나옵니다. 값이 크다고 인과가 성립하는 것도 아니어서, 두 변수를 동시에 움직이는 제3의 변수가 뒤에 있을 수 있습니다. 이 글의 기온 r≈0.85, 강수량 r≈0.12는 이 척도로 잰 숫자입니다.
다중공선성multicollinearity
회귀에 넣은 설명 변수 하나가 나머지 설명 변수들의 선형결합으로 거의 그대로 표현되는 상태입니다. 이렇게 되면 정규방정식의 XTX가 특이행렬에 가까워져 역행렬이 불안정해지고, 최소제곱 계수의 표준오차가 부풀어 표본이 조금만 달라져도 계수의 크기와 부호가 흔들립니다. 무너지는 것은 개별 계수의 해석이지 예측 정확도 자체는 아니라서, 같은 상관 구조가 유지되는 데이터에서는 예측만 놓고 보면 멀쩡할 수도 있습니다. 이 글에서 기온·습도·풍속이 서로 얽혀 있던 것이 그 경우였습니다.
VIFVariance Inflation Factor
변수 하나를 나머지 설명 변수 전부로 회귀시켜 얻은 결정계수 R²를 가지고 1/(1−R²)로 계산하는, 변수별 다중공선성 지표입니다. 값이 1이면 그 변수가 다른 변수들로 전혀 설명되지 않는다는 뜻이고, 10은 R²=0.9 즉 그 변수의 분산 90%가 나머지 변수로 설명된다는 뜻입니다. 다만 "10 초과" 기준은 통계적으로 유도된 임계값이 아니라 관행이라 분야에 따라 5를 쓰기도 합니다. 이 글에서는 기온·습도·풍속이 10을 넘겨 손볼 대상이 됐습니다.
파생변수derived feature
원시 열을 도메인 지식에 근거한 함수로 변환·결합해 새로 만든 입력 열입니다. 이 글의 불쾌지수 0.81T + 0.01·RH·(0.99T − 14.3) + 46.3처럼 기온과 습도 두 열을 의미가 분명한 한 열로 압축하면, 서로 겹치던 변수를 줄이는 동시에 사람이 냉방을 켜는 이유에 더 가까운 값을 모델에 넘기게 됩니다. 다만 예측 시점에 알 수 없는 값이나 정답 라벨을 재료로 쓰면 데이터 누수가 되어 검증 점수만 좋아지므로, 재료는 그 시점에 확보 가능한 열로 한정해야 합니다.

상관은 신호이자 함정

상관계수부터 뽑아 보니 기온은 전력사용량과 강한 상관(r≈0.85)이었고, 강수량은 거의 무관(r≈0.12)했습니다. 여기까지는 예상한 대로였는데, 표를 더 들여다보니 기온·습도·풍속이 서로도 강하게 상관하고 있었습니다. 다중공선성입니다. 이 상태로 변수를 다 넣으면 계수가 불안정해지고 해석도 어려워집니다.

# VIF로 다중공선성을 수치로 확인
from statsmodels.stats.outliers_influence import variance_inflation_factor
vif = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]
# temp, humidity, wind 이 10을 훌쩍 넘김 → 손봐야 한다

도메인 지식으로 압축한다

사람이 냉방을 켜는 건 온도가 높아서가 아니라 불쾌해서가 아닐까 하는 생각이 들었습니다. 그렇게 놓고 보니 서로 얽혀 있던 세 변수를 의미가 분명한 둘로 압축할 수 있었습니다.

# 불쾌지수(Thom) — 기온+습도를 하나로
df["discomfort"] = 0.81*T + 0.01*RH*(0.99*T - 14.3) + 46.3
# 체감온도 — 기온+풍속을 하나로
df["apparent"]  = T + 0.33*e - 0.70*wind - 4.0
df = df.drop(columns=["rain", "nonelec_cool", "solar"])  # 무의미·중복 제거

시간과 건물의 결

전력사용량에는 하루의 리듬이 있었습니다(22–05 저점, 10–16 고점). 모든 시각을 같은 값으로 취급하면 이 구조가 드러나지 않아서, 시간대와 건물을 축으로 분류해 각자의 고유 패턴을 모델이 볼 수 있게 했습니다.
