연구 회고: ETRI에서 만난 Vision AI — 법학도가 객체탐지로

2020년 2월• ETRI · Vision AI · CNN

ETRI 연구연수에서 제가 처음 맡은 일은 물류 드론이 찍은 영상 속 개인정보를 법적으로 어떻게 다뤄야 하는지 살피는 것이었습니다. 그런데 그 경계를 그으려면 기계가 무엇을 어떻게 식별하는가부터 알아야 했고, 자료를 따라가다 보니 객체탐지 코드를 붙잡고 있었습니다. 법학을 하던 제가 CNN과 객체탐지 쪽으로 방향을 넓히게 된 과정을 적어 둡니다.

먼저 짚고 가기
객체탐지Object Detection
이미지 안의 각 객체 인스턴스에 대해 위치(경계상자 좌표)와 클래스, 신뢰도를 함께 출력하는 컴퓨터 비전 과제입니다. 이미지 한 장에 라벨 하나를 붙이는 분류(classification)와 달리 '무엇'과 '어디에'를 동시에 답하며, 픽셀 단위 마스크를 내는 분할(segmentation)과도 구분됩니다. 드론 영상에서 개인정보를 가리려면 그 대상의 위치부터 확정해야 하므로 이 과제가 출발점이 됩니다.
합성곱 신경망CNN
작은 커널(필터)을 입력 위로 이동시키며 국소 영역과의 가중합(합성곱)을 반복하는 합성곱 계층으로 이루어진 신경망 구조입니다. 모든 위치에서 같은 커널의 가중치를 재사용(weight sharing)하고 국소 연결만 두어, 완전연결층보다 파라미터가 훨씬 적고 대상이 이동하면 반응 위치도 함께 이동하는 성질(translation equivariance)을 얻습니다. 본문의 conv2d 예제가 이 합성곱 연산 하나를 그대로 구현한 것입니다.
특징맵Feature Map
합성곱 계층이 하나의 필터를 모든 위치에 적용해 얻은 반응값(활성화)을 원래 공간 배치대로 담은 출력 배열입니다. 각 원소는 그 자리에서 필터가 찾는 무늬와 얼마나 맞는지를 나타내, 값이 클수록 해당 무늬가 강하게 나타난 위치입니다. 필터가 N개면 특징맵도 N장(채널) 생기며, 필터 자체가 아니라 필터를 통과시킨 결과라는 점에서 구분됩니다.
풀링Pooling
특징맵을 겹치지 않는 작은 창(예: 2×2, stride 2)으로 나눠 각 창을 대표값 하나로 줄이는 다운샘플링 연산입니다. 최대 풀링은 창 안의 최댓값을, 평균 풀링은 평균을 취하며, 공간 해상도를 낮추는 대신 작은 이동에 둔감한 성질(translation invariance)을 얻습니다. 학습되는 가중치가 없고 정확한 위치 정보를 버리기 때문에, 위치를 되살려야 하는 탐지에서는 이 손실이 다시 문제가 됩니다.
YOLO
이미지를 S×S 격자로 나눠 각 격자 칸이 경계상자와 신뢰도, 클래스 확률을 한 번의 순전파로 예측하는 단일 단계(one-stage) 객체탐지 모델 계열입니다. 이름은 "You Only Look Once"의 약자로, 영역을 먼저 제안한 뒤 분류하는 2단계 검출기(R-CNN 계열)와 달리 검출 전체를 한 번에 처리해 정확도를 일부 내주는 대신 실시간에 가까운 속도를 얻습니다. 본문에서 다루는 격자·앵커 이야기가 이 구조에서 출발합니다.

1. 법학도가 드론 프로젝트에 앉기까지

2020년부터 개발을 독학하던 시기에 ETRI의 Vision AI 연구연수에 참여했습니다. 처음 맡은 일은 기술보다 제도 쪽에 가까웠습니다. 물류 드론이 수집하는 영상에서 얼굴·번호판·사유지 같은 개인정보가 어떻게 노출되고 규율되어야 하는지를 법학의 언어로 정리하는 일이었고, 전공이 전공이다 보니 저에게는 익숙한 입구였습니다.

그런데 그 경계를 그으려면 드론 카메라가 실제로 무엇을 식별할 수 있는지부터 알아야 했습니다. "촬영된 정보"와 "식별 가능한 정보"는 다르고, 그 차이를 가르는 것은 모델의 성능이었습니다. 규범을 정리하려고 앉았다가 기술 문서를 펼쳤고, 그러다 코드 쪽으로 넘어가 있었습니다.

2. 개인정보 질문이 기술 질문으로

자료를 읽다 보니 두 질문이 사실상 같은 질문이었습니다. "이 프레임 안에 사람이 있는가, 번호판이 있는가?"는 법적으로는 개인정보 판단이지만, 기술적으로는 객체탐지 문제였습니다. 개인정보를 가리려면(마스킹) 먼저 찾아야(detection) 하니, 규제의 출발점과 검출의 출발점이 같은 자리에 있었습니다.

그러면서 제 무게중심도 "무엇을 금지할까"에서 "무엇을 어떻게 검출할까"로 옮겨 갔습니다. 이 시기에 학습 데이터를 다듬는 데이터 가공 일을 하면서 모델이 무엇을 먹고 무엇을 배우는지 조금씩 알게 됐습니다. 라벨 하나를 잘못 붙이면 검출이 어디서 흔들리는지가 눈에 들어왔고, 그 뒤로는 법 자료보다 기술 자료를 먼저 찾게 됐습니다.

3. 필터 — 이미지를 훑는 작은 창

제일 먼저 붙든 개념이 합성곱(convolution)이었습니다. 이미지의 모든 픽셀을 완전연결로 잇는 방식은 파라미터가 지나치게 늘어나고, 대상이 몇 픽셀만 움직여도 학습한 것이 잘 맞지 않습니다. CNN은 대신 작은 필터 하나를 이미지 전체에 미끄러뜨리며 같은 가중치를 재사용합니다. 필터는 특정 무늬(세로 경계, 모서리)에 반응하는 작은 창이고, 그 반응을 위치마다 기록한 결과가 특징맵입니다.

이렇게 하는 이유는 찾는 대상이 이미지 어디에나 나타날 수 있기 때문입니다. 왼쪽 위의 경계든 오른쪽 아래의 경계든 같은 필터 하나로 잡는 편이 효율적입니다. 설명만 읽을 때는 잘 와닿지 않아서, 세로 경계에 반응하는 필터를 numpy로 직접 밀어 봤습니다.

import numpy as np

# 3×3 세로 경계 검출 필터 (Sobel의 뼈대)
kernel = np.array([[-1, 0, 1],
                   [-2, 0, 2],
                   [-1, 0, 1]])

def conv2d(img, k):
    kh, kw = k.shape
    oh, ow = img.shape[0] - kh + 1, img.shape[1] - kw + 1
    out = np.zeros((oh, ow))
    for y in range(oh):
        for x in range(ow):
            patch = img[y:y+kh, x:x+kw]        # 커널만 한 창을 떼어
            out[y, x] = np.sum(patch * k)      # 겹친 자리끼리 곱해 더한다
    return out

img = np.array([[0, 0, 0, 9, 9, 9],
                [0, 0, 0, 9, 9, 9],
                [0, 0, 0, 9, 9, 9]], dtype=float)  # 왼쪽 어둡고 오른쪽 밝은 면
print(conv2d(img, kernel))    # [[0 36 36 0]] — 평평한 면은 0, 경계에서만 반응

이 출력이 특징맵입니다. 평평한 면에서는 0이고 세로 경계가 지나는 자리에서만 큰 값이 나오는데, 필터가 자기가 찾는 무늬를 만난 지점만 밝힌 것입니다. 학습은 이 필터의 숫자들을 데이터로부터 맞춰 가는 과정이라는 설명이 그제야 눈에 들어왔습니다.

4. 특징맵과 풀링 — 위치를 흐리게

특징맵을 그대로 다음 층에 넘기면 여전히 크고, 미세한 위치에 지나치게 예민합니다. 풀링(pooling)은 작은 창에서 가장 센 반응만 남겨 크기를 줄이고, 덤으로 작은 이동에 둔감해지게 만듭니다. 고양이가 몇 픽셀 움직여도 가장 강한 고양이 반응은 그대로 살아남기 때문입니다.

위치를 흐리게 만드는 편이 이득인 이유는 분류에서는 "무엇이 있나"가 중요하고 "정확히 몇 픽셀에"는 덜 중요하기 때문입니다. 2×2 최대 풀링을 손으로 짜 보고 나서야 이 절충이 무엇을 버리고 무엇을 얻는지 정리가 됐습니다.

def max_pool(fmap, size=2, stride=2):
    oh = (fmap.shape[0] - size) // stride + 1
    ow = (fmap.shape[1] - size) // stride + 1
    out = np.zeros((oh, ow))
    for y in range(oh):
        for x in range(ow):
            patch = fmap[y*stride:y*stride+size, x*stride:x*stride+size]
            out[y, x] = patch.max()            # 창 안에서 가장 센 반응만 남긴다
    return out

fmap = np.array([[1, 3, 2, 1],
                 [4, 6, 1, 0],
                 [0, 1, 5, 2],
                 [1, 2, 3, 4]], dtype=float)
print(max_pool(fmap))         # [[6 2] [2 5]] — 크기 절반, 강한 특징만 생존
*

여기서 나중에 걸리는 지점이 하나 생깁니다. 풀링은 정확한 위치를 일부러 흐리는데, 분류에서는 그게 도움이 됩니다. 반면 탐지는 "무엇이 어디에"를 답해야 하니 풀링이 버린 위치를 다시 세워야 합니다. 뒤에 나올 YOLO의 격자와 앵커도 이 문제에서 출발합니다.

5. 밑바닥부터 짜 보고서야 보였다

개념은 알겠는데 손에 잡히지 않아 붙든 책이 사이토 고키의 『밑바닥부터 시작하는 딥러닝』이었습니다. 프레임워크 없이 numpy만으로 퍼셉트론부터 오차역전파, CNN까지 직접 쌓아 올리는 책입니다. 계층 하나하나를 forward·backward를 가진 클래스로 구현해 나가다 보니, 딥러닝이 미분과 행렬곱의 조립이라는 감각이 조금 생겼습니다.

CNN을 다루는 장에서 오래 붙잡았던 건 im2col이었습니다. 앞의 conv2d처럼 반복문으로 창을 훑으면 느린데, 각 창을 한 줄로 펼쳐 쌓으면 합성곱이 행렬 한 번 곱하기로 바뀝니다. 글로 읽을 때는 잘 그려지지 않아서 짧게 짜 봤습니다.

# 반복문으로 창을 훑는 대신, 창마다 한 줄로 펼쳐 쌓는다 (im2col)
def im2col(img, kh, kw):
    oh, ow = img.shape[0] - kh + 1, img.shape[1] - kw + 1
    cols = [img[y:y+kh, x:x+kw].flatten()      # 창 하나 = 한 줄
            for y in range(oh) for x in range(ow)]
    return np.array(cols)                        # (창 개수, kh*kw)

col = im2col(img, 3, 3)         # 3×6 이미지, 3×3 커널 → (4, 9)
out = col @ kernel.flatten()    # 합성곱이 '행렬 한 번 곱하기'가 된다
print(out.reshape(1, 4))        # [[0 36 36 0]] — conv2d와 같은 결과, 반복문은 사라졌다

결과는 3절의 conv2d와 같은데 반복문은 사라졌습니다. 이 책이 제게 남긴 건 특정 수식보다 태도에 가까웠습니다. 라이브러리의 Conv2D 한 줄 뒤에서 무엇이 도는지 알고 부르는 것과 모르고 부르는 것은 달랐습니다. ETRI에서 남이 짠 파이프라인을 손볼 때도 이 감각 덕분에 어디를 만져야 할지 짐작할 수 있었습니다.

6. 탐지로, 그리고 YOLO로

분류(이미지 한 장 = 라벨 하나)에서 탐지(어디에 무엇이 몇 개)로 넘어가면, 풀링이 흐려 버린 위치를 다시 세워야 합니다. ETRI에서 익힌 필터 → 특징맵 → 풀링이라는 문법은 그 뒤로 검출을 마주할 때마다 기준점이 됐습니다. 대흥정보에서 YOLO로 주차장 차량을 감지하는 일을 할 때도, 나중에 YOLO 원 논문을 다시 읽을 때도 그랬습니다.

격자가 왜 필요한지, 앵커가 무엇을 보정하는지도 "합성곱은 위치에 강하고 풀링은 위치를 흐린다"는 이 지점에서 출발합니다. 기초 문법을 손으로 한 번 세워 두니 검출 모델의 낯선 설계도 처음 보는 그림이 아니라 아는 문장의 변형으로 읽혔습니다. 개인정보라는 법적 물음에서 시작한 길이 여기까지 이어졌습니다.

*

돌아보면 방향을 바꾼 데에 특별한 결심이 있었던 것은 아닙니다. "드론이 무엇을 식별하는가"라는 질문을 놓지 못하고 따라갔더니 코드 앞에 앉아 있었습니다. 새 목표를 세웠다기보다 하나의 질문을 오래 붙들고 있었던 쪽에 가깝습니다.

7. 정리 — 전공이 아니라 질문이 옮긴다

정리해 보면 저를 옮긴 것은 전공이 아니라 질문이었습니다. 드론이 무엇을 보는가라는 법적 질문이 기계가 무엇을 보는가라는 기술 질문으로 이어졌고, 그 답을 찾다가 개발 쪽에 자리를 잡았습니다. ETRI에서의 시간이 성과 목록으로 남지는 않았지만, 방향을 바꾼 계기로는 지금도 또렷하게 기억합니다.

  • 질문이 경계를 넘는다. 개인정보라는 법적 물음과 객체탐지라는 기술 물음은 한 질문의 양면이었다. 무언가를 가리려면 먼저 찾아야 하기 때문이다.
  • 개념은 손으로 짤 때 잡힌다. 필터·풀링·im2col을 numpy로 직접 짜 보고서야 프레임워크 한 줄이 읽혔다. 그 감각 덕분에 남이 짠 파이프라인도 열어 볼 수 있었다.
  • 기초 문법은 오래 남는다. 필터→특징맵→풀링은 YOLO 차량감지까지 그대로 기준점이 됐다. 새 모델은 낯선 그림이 아니라 아는 문장의 변형이었다.
