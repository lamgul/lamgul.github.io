연구 회고: ETRI에서 만난 Vision AI — 법학도가 객체탐지로

2020년 2월• ETRI · Vision AI · CNN

드론이 하늘에서 무엇을 볼 수 있는가 — ETRI 연구연수는 그 물음에서 시작됐습니다. 물류 드론이 찍은 영상 속 개인정보를 어떻게 다뤄야 하는가, 법 쪽에서 던진 질문이었죠. 그런데 답하려면 결국 기계가 무엇을 어떻게 식별하는가를 알아야 했습니다. 법학도였던 제가 객체탐지 코드 앞에 앉게 된 계기를 돌아봅니다.

먼저 짚고 가기
객체탐지Object Detection
사진 속에 무엇이 어디에 몇 개 있는지를 찾아내는 기술입니다. "이 사진은 고양이"라고만 답하는 분류와 달리, "왼쪽 위에 사람, 오른쪽에 번호판" 하고 위치까지 짚습니다. 드론 영상에서 개인정보를 가리려면 먼저 그 대상을 찾아야 하니, 이 글의 출발점입니다.
합성곱 신경망CNN
작은 돋보기 같은 필터를 이미지 위로 미끄러뜨리며 무늬를 훑어 특징을 뽑는 신경망입니다. 픽셀 하나하나를 전부 잇는 대신 같은 필터를 재사용해, 대상이 화면 어디에 있어도 적은 계산으로 잡아냅니다. 오늘날 이미지 AI의 기본 뼈대입니다.
특징맵Feature Map
필터가 이미지를 훑고 남긴 반응 지도입니다. 필터가 찾는 무늬(세로 경계 등)를 만난 자리는 밝게, 못 만난 자리는 어둡게 표시됩니다. 어디에 무엇이 있는지에 대한 힌트가 여기에 담깁니다.
풀링Pooling
특징맵을 작은 구역으로 나눠 가장 센 반응만 남기며 크기를 줄이는 과정입니다. 사진을 살짝 축소하듯 미세한 위치는 흐려지지만, 대상이 몇 픽셀 움직여도 흔들리지 않게 해 줍니다. 뒤에서 탐지와 부딪치는 긴장의 원인이기도 합니다.
YOLO
이미지를 격자로 나눠 한 번에 객체의 위치와 종류를 예측하는 대표적인 객체탐지 모델입니다. 이름은 "You Only Look Once"의 약자로, 빠른 속도가 강점이라 실시간 영상에 자주 쓰입니다.

1. 법학도가 드론 프로젝트에 앉기까지

2020년부터 개발을 독학하던 시기, 저는 ETRI의 Vision AI 연구연수에 발을 들였습니다. 처음 맡은 결은 기술이 아니라 제도 쪽이었습니다. 물류 드론이 수집하는 영상에서 얼굴·번호판·사유지 같은 개인정보가 어떻게 노출되고 규율되어야 하는가 — 법학의 언어로 접근하는 문제였죠. 배경이 배경인지라 저에게는 익숙한 입구였습니다.

문제는 그 경계를 그으려면 드론 카메라가 실제로 무엇을 식별할 수 있는지부터 알아야 했다는 것입니다. "촬영된 정보"와 "식별 가능한 정보"는 다르고, 그 차이를 가르는 건 결국 모델의 성능입니다. 규범을 말하려다 기술 문서를 펼쳤고, 어느새 코드 쪽으로 넘어가 있었습니다.

2. 개인정보 질문이 기술 질문으로

어느 순간 두 질문이 같은 질문이라는 게 보였습니다. "이 프레임 안에 사람이 있는가, 번호판이 있는가?"는 법적으로는 개인정보 판단이지만, 기술적으로는 정확히 객체탐지 문제였습니다. 개인정보를 가리려면(마스킹) 먼저 찾아야(detection) 하니, 규제의 출발점이 곧 검출의 출발점이었던 셈입니다.

그 지점에서 제 무게중심이 "무엇을 금지할까"에서 "무엇을 어떻게 검출할까"로 넘어갔습니다. 이 시기에 학습 데이터를 다듬는 데이터 가공 일을 하면서, 모델이 무엇을 먹고 무엇을 배우는지 몸으로 알게 됐습니다. 라벨 하나를 잘못 붙이면 검출이 어디서 흔들리는지가 눈에 들어오기 시작했고, 그때부터는 되돌아갈 수 없었습니다.

3. 필터 — 이미지를 훑는 작은 창

가장 먼저 넘어야 할 개념이 합성곱(convolution)이었습니다. 이미지의 모든 픽셀을 완전연결로 잇는 방식은 파라미터가 폭발하고, 대상이 몇 픽셀만 움직여도 무너집니다. CNN은 대신 작은 필터 하나를 이미지 전체에 미끄러뜨리며 같은 가중치를 재사용합니다. 필터는 특정 무늬(세로 경계, 모서리)에 반응하는 작은 창이고, 그 반응을 위치마다 기록한 결과가 특징맵입니다.

왜 이렇게 하냐면 — 찾는 것이 이미지 어디에나 나타날 수 있기 때문입니다. 왼쪽 위의 경계든 오른쪽 아래의 경계든 같은 필터 하나로 잡아야 효율적이죠. 세로 경계에 반응하는 필터를 numpy로 직접 밀어 보면 그 감각이 잡힙니다.

import numpy as np

# 3×3 세로 경계 검출 필터 (Sobel의 뼈대)
kernel = np.array([[-1, 0, 1],
                   [-2, 0, 2],
                   [-1, 0, 1]])

def conv2d(img, k):
    kh, kw = k.shape
    oh, ow = img.shape[0] - kh + 1, img.shape[1] - kw + 1
    out = np.zeros((oh, ow))
    for y in range(oh):
        for x in range(ow):
            patch = img[y:y+kh, x:x+kw]        # 커널만 한 창을 떼어
            out[y, x] = np.sum(patch * k)      # 겹친 자리끼리 곱해 더한다
    return out

img = np.array([[0, 0, 0, 9, 9, 9],
                [0, 0, 0, 9, 9, 9],
                [0, 0, 0, 9, 9, 9]], dtype=float)  # 왼쪽 어둡고 오른쪽 밝은 면
print(conv2d(img, kernel))    # [[0 36 36 0]] — 평평한 면은 0, 경계에서만 반응

출력이 곧 특징맵입니다. 평평한 면에서는 0이고 세로 경계가 지나는 자리에서만 큰 값이 뜹니다 — 필터가 자기가 찾는 무늬를 만난 지점만 밝힌 것이죠. 학습이란 이 필터의 숫자들을 데이터로부터 스스로 맞춰 가는 과정일 뿐입니다.

4. 특징맵과 풀링 — 위치를 흐리게

특징맵을 그대로 다음 층에 넘기면 여전히 크고, 미세한 위치에 지나치게 예민합니다. 풀링(pooling)은 작은 창에서 가장 센 반응만 남겨 크기를 줄이고, 덤으로 작은 이동에 둔감해지게 만듭니다. 고양이가 몇 픽셀 움직여도 "가장 강한 고양이 반응"은 그대로 살아남으니까요.

왜 흐리게 만드는 게 이득일까요. 분류에서는 "무엇이 있나"가 중요하지 "정확히 몇 픽셀에"는 덜 중요하기 때문입니다. 2×2 최대 풀링을 손으로 짜 보면 이 절충이 뭘 버리고 뭘 얻는지 분명해집니다.

def max_pool(fmap, size=2, stride=2):
    oh = (fmap.shape[0] - size) // stride + 1
    ow = (fmap.shape[1] - size) // stride + 1
    out = np.zeros((oh, ow))
    for y in range(oh):
        for x in range(ow):
            patch = fmap[y*stride:y*stride+size, x*stride:x*stride+size]
            out[y, x] = patch.max()            # 창 안에서 가장 센 반응만 남긴다
    return out

fmap = np.array([[1, 3, 2, 1],
                 [4, 6, 1, 0],
                 [0, 1, 5, 2],
                 [1, 2, 3, 4]], dtype=float)
print(max_pool(fmap))         # [[6 2] [2 5]] — 크기 절반, 강한 특징만 생존
*

여기 나중에 발목을 잡는 긴장이 숨어 있습니다. 풀링은 정확한 위치를 일부러 흐립니다 — 그게 분류에는 미덕이죠. 그런데 탐지는 "무엇이 어디에"를 답해야 하니, 풀링이 버린 위치를 되찾아야 합니다. 이 모순이 뒤에 나올 YOLO의 격자·앵커가 풀려던 바로 그 문제입니다.

5. 밑바닥부터 짜 보고서야 보였다

개념은 알겠는데 손에 잡히지 않던 시기, 붙든 책이 사이토 고키의 『밑바닥부터 시작하는 딥러닝』이었습니다. 프레임워크 없이 numpy만으로 퍼셉트론부터 오차역전파, CNN까지 직접 쌓아 올리는 책입니다. 계층 하나하나를 forward·backward를 가진 클래스로 구현하는 구성이라, 딥러닝이 마법이 아니라 미분과 행렬곱의 조립이라는 감각을 줍니다.

CNN을 다루는 장의 핵심은 im2col이라는 트릭이었습니다. 앞의 conv2d처럼 반복문으로 창을 훑으면 느립니다. 대신 각 창을 한 줄로 펼쳐 쌓으면, 합성곱이 행렬 한 번 곱하기로 바뀝니다. 그 책이 왜 이 방식을 강조하는지는 직접 짜 보면 바로 납득됩니다.

# 반복문으로 창을 훑는 대신, 창마다 한 줄로 펼쳐 쌓는다 (im2col)
def im2col(img, kh, kw):
    oh, ow = img.shape[0] - kh + 1, img.shape[1] - kw + 1
    cols = [img[y:y+kh, x:x+kw].flatten()      # 창 하나 = 한 줄
            for y in range(oh) for x in range(ow)]
    return np.array(cols)                        # (창 개수, kh*kw)

col = im2col(img, 3, 3)         # 3×6 이미지, 3×3 커널 → (4, 9)
out = col @ kernel.flatten()    # 합성곱이 '행렬 한 번 곱하기'가 된다
print(out.reshape(1, 4))        # [[0 36 36 0]] — conv2d와 같은 결과, 반복문은 사라졌다

결과는 3절의 conv2d와 완전히 같은데 반복문은 증발했습니다. 이 책이 제게 남긴 건 특정 수식이 아니라 태도였습니다 — 라이브러리의 Conv2D 한 줄 뒤에 무엇이 돌아가는지 알고 부르는 것과 모르고 부르는 것은 다릅니다. ETRI에서 남이 짠 파이프라인을 손볼 때, 이 밑바닥 감각이 "어디를 만져야 하는지"를 알려 줬습니다.

6. 탐지로, 그리고 YOLO로

분류(이미지 한 장 = 라벨 하나)에서 탐지(어디에 무엇이 몇 개)로 넘어가면, 풀링이 흐려 버린 위치를 다시 세워야 합니다. ETRI에서 익힌 필터 → 특징맵 → 풀링이라는 문법은 그 뒤로 검출을 마주할 때마다 제 기준점이 됐습니다. 대흥정보에서 YOLO로 주차장 차량을 감지하는 일을 하든, 나중에 YOLO 원 논문을 다시 뜯어보든 마찬가지였습니다.

격자가 왜 필요한지, 앵커가 무엇을 보정하는지는 결국 "합성곱은 위치에 강하고 풀링은 위치를 흐린다"는 이 지점에서 출발합니다. 기초 문법을 손으로 한 번 세워 두니, 검출 모델의 낯선 설계도 처음 보는 그림이 아니라 익숙한 문장의 변형으로 읽혔습니다. 개인정보라는 법적 물음에서 시작한 길이 여기까지 이어진 셈입니다.

*

돌이켜보면 전향은 극적인 결심이 아니라 질문을 끝까지 따라간 결과였습니다. "드론이 무엇을 식별하는가"를 놓지 못했고, 그 답을 좇다 보니 코드 앞이었습니다. 각도를 바꾼 건 새 목표가 아니라 오래 붙든 질문이었죠.

7. 정리 — 전공이 아니라 질문이 옮긴다

회고해 보면 저를 옮긴 건 전공이 아니라 질문이었습니다. 드론이 무엇을 보는가라는 법적 질문이 기계가 무엇을 보는가라는 기술 질문으로 이어졌고, 그 답을 좇다 개발자가 됐습니다. ETRI에서의 시간은 성과의 목록으로 남지 않았지만, 방향을 바꾼 계기로는 또렷합니다.

  • 질문이 경계를 넘는다. 개인정보라는 법적 물음과 객체탐지라는 기술 물음은 한 질문의 양면이었다 — 무언가를 가리려면 먼저 찾아야 하니까.
  • 개념은 손으로 짤 때 잡힌다. 필터·풀링·im2col을 numpy로 직접 짜 보고서야 프레임워크 한 줄이 읽혔다. 밑바닥 감각이 남의 파이프라인을 고칠 자신을 줬다.
  • 기초 문법은 오래 남는다. 필터→특징맵→풀링은 YOLO 차량감지까지 그대로 기준점이 됐다. 새 모델은 낯선 그림이 아니라 아는 문장의 변형이었다.
