가변 폰트로 UI에 숨을 불어넣기

2025년 2월• 약 6분• CSS · Variable Fonts

가변 폰트(variable font)는 파일 하나에 수천 개의 글꼴을 접어 넣습니다. 얇은 것과 굵은 것 사이의 모든 중간값이 연속적으로 들어 있죠. 이걸 CSS로 연속적으로 움직일 수 있다는 걸 알고 나니, 도구부터 만들고 싶어졌습니다. 그렇게 나온 게 활자 실험실입니다.

한 줄이 전부를 리셋한다

가변 폰트의 축은 font-variation-settings 한 줄로 제어합니다. 그런데 여기 함정이 있습니다. 이 속성은 명시하지 않은 축을 전부 기본값으로 되돌립니다. 무게만 바꾸려고 이렇게 쓰면—

el.style.fontVariationSettings = '"wght" 700';
// 방금 -6도로 기울여 둔 slant가... 0으로 돌아간다

—직전에 조절한 기울기가 조용히 사라집니다. CSS 커스텀 속성으로 축을 쪼개 관리하는 방법도 있지만, 저는 매 렌더마다 다섯 축을 통째로 직렬화하는 쪽을 택했습니다. 상태는 JS가 들고, CSS엔 완성된 문자열만 넘깁니다. 단순하고, 틀릴 구석이 없습니다.

모든 축이 연속인 건 아니다

Recursive 폰트에는 필기체로 바꾸는 CRSV 축이 있는데, 슬라이더를 연결하고 한참을 "왜 중간에서 안 변하지?" 하며 헤맸습니다. 스펙을 읽고서야 알았습니다. 이 축은 0과 1만 의미가 있는 이산(discrete) 축이었던 겁니다. 그래서 슬라이더는 연속으로 두되, 적용할 땐 0.5를 기준으로 스냅했습니다.

디자인 축에도 자료형이 있다. 연속인 줄 알았던 게 사실은 불리언일 수 있다.

슬라이더 열 개보다 손끝 하나

축마다 슬라이더를 다는 건 정직하지만 지루합니다. 가장 자주 함께 만지는 두 축 — 무게와 기울기 — 은 2D 패드로 묶었습니다. 가로로 끌면 굵어지고, 세로로 끌면 기울어집니다. 좌표를 축의 범위로 선형 매핑하는 게 전부인데, 손가락 하나로 두 값을 동시에 잡는 감각은 슬라이더 두 개와 완전히 다릅니다. 입력의 차원을 줄이면 표현의 차원이 늘어납니다.

프리셋은 점프가 아니라 이동

가장 좋아하는 디테일입니다. 프리셋 버튼을 누르면 값이 툭 바뀌는 대신, 현재 상태에서 목표 상태로 다섯 축이 동시에 미끄러집니다.

const ease = (t) => 1 - Math.pow(1 - t, 3);   // ease-out cubic
keys.forEach((k) => state[k] = from[k] + (target[k] - from[k]) * ease(p));

560ms 동안 '그로테스크'에서 '손글씨'로 글자가 흘러가는 모습은, 정적인 폰트로는 절대 만들 수 없는 순간입니다. 애니메이션의 대상이 위치나 색이 아니라 글자의 형태 그 자체라는 게 여전히 신기합니다.

성능과 남은 빚

입력할 때마다 폰트를 리레이아웃하지만, 단일 요소라 60fps가 유지됩니다. 문단이 길어지면 content-visibility가 필요해질 겁니다. 미리보기를 contenteditable로 만든 탓에 붙여넣기 서식이 딸려 오는 문제도 남아 있습니다. plaintext-only의 크로스브라우징이 아직 애매해서, 알고 남겨둔 빚으로 두었습니다.

결국 이 도구를 만든 건 폰트를 읽기 위해서였습니다. 축을 직접 만져보고 나서야 "이 폰트는 왜 이 축을 넣었을까"가 이해됐거든요. 좋은 도구는 종종 대상을 더 잘 보게 해주는 렌즈입니다.
