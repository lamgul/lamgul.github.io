무명MUMYEONG SERIES

이름을 부르면 병드는 서울(무명록), 검은 비가 기억을 지우는 밤(무명비). 같은 세계관 공식으로 만든 두 게임과 그 제작 파이프라인.

흰비가 지나간 서울에서는 이름을 부르면 병이 옮고(무명록), 검은 비가 내리는 밤에는 이름과 기억이 지워집니다(무명비: 영만의 밤). 서로 다른 두 게임이지만, 한 저장소에 정리해 둔 같은 세계관 공식에서 출발했습니다. 위 데모는 그 공식과 두 게임의 뼈대를 브라우저 안에 다시 세운 것입니다.

하나의 공식, 두 개의 밤

시리즈의 시작점은 게임이 아니라 코드가 한 줄도 없는 레포입니다. 세계관 문서와 일러스트만 담은 재료 창고가 따로 있고, 배경을 만드는 공식을 이렇게 못 박아 둡니다.

익숙한 한국 장소 + 비현실 규칙 + 생활 적응 + 통제 세력 + 작은 선택의 대가

장소 10종, 이상 현상 9종, 생활 소품 10종, 세력 9종, 금기 8종이 소재 라이브러리 표로 정리되어 있고, 게임마다 여기서 하나씩 뽑아 변주합니다. 무명록은 이름·기록·장부를, 무명비는 비·기억·별명을 가져갔습니다. 캐릭터는 이름 풀만 공유하고 성격·나이·관계는 게임마다 다시 만듭니다 — 정본 설정을 확정하지 않는 것이 이 창고의 운영 원칙입니다. 데모의 세계관 생성기는 그 표들을 그대로 이식해, 같은 공식으로 다음 게임 후보를 뽑아 봅니다.

엔진의 문법: 장면 테이블과 상태 객체

두 게임 모두 프레임워크 없는 단일 HTML이고, 서버도 빌드도 없습니다. 구조도 같습니다. 회차의 모든 것이 상태 객체 하나(몸·정신·호명 같은 스탯, 단서, 물자, 습관, 관계)에 담기고, 이야기는 SCENES 테이블의 장면과 {text, note, req, to, effect} 선택지로 굴러갑니다. req가 잠긴 선택지는 숨기지 않고 잠긴 채로 보여 줍니다 — 무엇을 놓쳤는지 보이는 것이 리플레이의 동력이 되기 때문입니다.

재미있는 설계 결정 하나는 무명록의 확장팩 로딩입니다. 84KB짜리 코어 엔진 뒤에 확장 스크립트 7개가 붙는데, 각 파일은 전역 API 하나로 장면을 주입합니다. 원작 코드에서 그대로 가져오면 이렇습니다.

// 게임/소스/게임.js — 확장팩 등록 API (일부 생략)
window.무명록_확장등록 = (pack) => {
  PENDING_EXTENSIONS.push(pack);
  if (contentReady) applyExtensionPack(pack);
};

function applyExtensionPack(pack) {
  if (!pack || pack.applied) return;
  if (pack.scenes) Object.assign(SCENES, pack.scenes);
  if (pack.sceneChoices) {
    Object.entries(pack.sceneChoices).forEach(([sceneId, choices]) => {
      if (SCENES[sceneId]) SCENES[sceneId].choices.push(...choices);
    });
  }
  pack.applied = true;
}

덕분에 코어는 확장 파일의 존재를 모르고, 확장은 로드 순서를 몰라도 됩니다(먼저 도착하면 대기열에 쌓입니다). 장편 1~3부, 관계 장면, 서브스토리가 전부 이 좁은 문 하나로 들어옵니다. 호명은 이 세계관의 심장이라 규칙도 코드에 그대로 있습니다. 첫 번째는 기억, 두 번째는 그리움, 세 번째는 병 — 게이지가 3에 닿으면 어떤 장면으로 가던 중이든 위기가 끼어듭니다.

// 게임/소스/게임.js — checkCrisis()
if (S.call >= 3 && !String(nextScene).startsWith("ending") && nextScene !== "call_crisis") {
  S.pendingScene = nextScene;
  return "call_crisis";
}

데모가 재현한 것, 생략한 것

  • 재현 — 무명록 프롤로그~2장 초입 5장면. 장면·선택지·효과 텍스트를 원작 데이터에서 그대로 발췌했습니다.
  • 재현 — 스탯 미터, 단서 칩, 조건부 잠금 선택지, 호명 경고와 E7 '세 번째 호명' 위기 분기. 요약판에서는 호명 3에 닿기 어려워 '시연 · 호명 +1' 버튼을 따로 두었습니다.
  • 재현 — 소재 라이브러리 46개 요소를 이식한 세계관 생성기. 요소별 재추첨이 가능합니다.
  • 생략 — localStorage 자동 저장과 기록실, 인카운터 풀 대부분, 관계(신뢰)·능력 시스템, 엔딩 11종 중 E7 외의 분기.
  • 생략 — 캐릭터 일러스트 64장. 원천 자산은 비공개로 둔다는 저장소 원칙을 따랐습니다.
