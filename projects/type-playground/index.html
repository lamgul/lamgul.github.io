폰트 파일을 하나 열어봤다가 시작된 프로젝트입니다. Recursive 한 파일 안에 weight, slant, casual, monospace, cursive 다섯 개의 축이 들어 있었고, 이걸 코드로 연속적으로 움직일 수 있다는 게 신기해서 도구부터 만들었습니다.


      
가변 폰트는 어떻게 조종하나

      
브라우저에서 가변 폰트의 축은 font-variation-settings 한 줄로 제어됩니다. 문제는 이 속성이 명시하지 않은 축을 전부 기본값으로 되돌린다는 점이었습니다. weight만 바꾸려고 "wght" 700만 넣으면, 방금 조절한 slant가 0으로 리셋됩니다. 그래서 매 프레임 다섯 축을 전부 직렬화해서 통째로 씁니다.

      
// 명시 안 한 축은 기본값으로 리셋된다 → 항상 다섯 개를 함께 쓴다
preview.style.fontVariationSettings =
  `"wght" ${wght}, "slnt" ${slnt}, "CASL" ${casl}, "MONO" ${mono}, "CRSV" ${crsv}`;


      
슬라이더보다 손맛

      
축마다 슬라이더를 다는 건 지루했습니다. 가장 자주 함께 만지는 두 축 — 무게와 기울기 — 은 2D 패드로 묶었습니다. 가로로 끌면 굵어지고 세로로 끌면 기울어집니다. 좌표를 축 범위로 매핑하는 게 전부지만, 손가락 하나로 두 값을 동시에 잡는 감각이 완전히 다릅니다.

      
*
CRSV(필기체)는 0과 1 사이 중간값이 의미가 없는 이산 축이라, 슬라이더는 연속으로 두되 실제 적용할 땐 0.5를 기준으로 0/1로 스냅했습니다. 스펙을 안 읽었으면 "왜 중간에서 안 변하지?"로 한참 헤맸을 부분입니다.


      
프리셋은 '점프'가 아니라 '이동'

      
프리셋 버튼을 누르면 값이 툭 바뀌는 대신, 현재 상태에서 목표 상태로 560ms 동안 트윈됩니다. 다섯 축이 동시에 미끄러지며 '그로테스크'에서 '손글씨'로 넘어가는 모습이, 이 도구에서 제일 좋아하는 순간입니다. easing은 1 - (1-t)³ — 흔한 ease-out 큐빅입니다.


      
남은 것 · 배운 것

      
    
        
  • contenteditable의 함정. 미리보기를 직접 편집하게 만들었더니 붙여넣기 시 서식이 딸려 들어왔습니다. plaintext-only가 아직 크로스브라우징이 애매해서, 지금은 감수하고 두었습니다. 알고 남겨둔 빚입니다.
    • 
        
  • 성능. 매 입력마다 폰트를 리레이아웃하지만, 단일 요소라 60fps가 유지됩니다. 문단이 길어지면 content-visibility가 필요해질 겁니다.
    • 
        
  • 축을 손으로 만져보고 나서야 "이 폰트는 왜 이 축을 넣었나"가 이해됐습니다. 결국 도구를 만든 건 폰트를 읽기 위해서였습니다.
