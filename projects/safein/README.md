# 세이프인 — SafeIn

코드엣지에서 백엔드를 단독 설계·운영 중인 산업안전 시스템의 핵심 흐름을
관제 콘솔(좌) + 375px 근로자 앱(우) 한 화면의 정적 데모로 재현했다.
관제에서 TBM·공지를 발행하면 근로자 앱에 선택한 언어로 도착하고,
앱의 '확인'이 콘솔의 실시율·안전점수가 된다.

**Live:** https://lamgul.github.io/projects/safein/
**사례 연구:** https://lamgul.github.io/cases/safein.html

## 재미있었던 지점

- **구조는 실제, 데이터는 가공.** 도메인 모양(TBM 발행→참여자 확인, 점검 판정→지적→조치,
  공지→번역→피드)과 `POST /app/tbms/:id/confirm` 같은 엔드포인트 형태는 실제 제품을 따랐다.
  화면의 현장("성수 물류센터 신축현장")·수치·문안은 전부 이 데모를 위해 새로 지어냈다.
- **사전 번역 목데이터 6개 언어.** 한국어·English·中文·Tiếng Việt·ភាសាខ្មែរ·Русский.
  언어를 바꾸면 TBM·공지 콘텐츠와 앱 UI 문자열까지 즉시 전환된다.
  실제 제품은 온프레미스 vLLM이 15개 언어를 생성한다 — 데모는 그 결과를 흉내 낼 뿐이다.
- **양방향 동기화.** 인메모리 스토어 하나를 콘솔과 앱이 같이 구독한다.
  발행 → 앱 피드 도착(NEW 플래시), 확인 → 실시율·안전점수 갱신.
- **localStorage가 PostgreSQL 대신.** `safein:demo:v1` 키에 상태가 남고, '초기화'로 시드로 되돌린다.
- **네트워크 요청 0.** 폰트 CDN을 제외하면 전부 이 폴더 안에서 끝난다.

`safein.js` 한 파일이 콘솔·앱·스토어·번역 사전 전부다.
