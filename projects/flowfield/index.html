흐름Flowfield

쓸모를 잠시 내려놓고 만든 것. 노이즈로 그린 보이지 않는 벡터장 위를 수천 개의 입자가 흐릅니다. 화면을 움직이거나 누르면 흐름이 손끝을 피해 갈라집니다.

역할그냥, 즐거움
스택Canvas 2D · 값 노이즈 · 의존성 0
코드GitHub ↗
flowfield — canvas 2d, 배치 스트로크
0 입자 · 화면을 움직이거나 눌러 보세요

모든 걸 쓸모로 재던 어느 주에, 일부러 쓸모없는 걸 만들고 싶었습니다. 결과물은 이 흐름장(flow field)입니다. 그런데 예쁜 걸 60fps로 유지하는 건 생각보다 엔지니어링이었습니다.

무작위가 아니라 '노이즈'

입자를 그냥 랜덤으로 움직이면 지저분한 모래알이 됩니다. 흐름이 생기려면 이웃한 지점의 방향이 서로 비슷해야 합니다. 그래서 값 노이즈(value noise)를 직접 짰습니다. 격자마다 해시로 값을 뽑고, 그 사이를 부드럽게 보간해 연속적인 각도장을 만듭니다. 라이브러리를 쓰면 한 줄이지만, 해시 한 함수가 어떻게 '결'을 만드는지 보고 싶었습니다.

const a = noise(p.x * scale + drift.x, p.y * scale + drift.y) * TAU * 2;
p.x += Math.cos(a) * speed;   // 각도장을 따라 흐른다
p.y += Math.sin(a) * speed;

수천 번의 선, 몇 번의 draw call

매 프레임 수천 개의 선을 긋습니다. 입자마다 stroke()를 부르면 GPU가 숨을 헐떡입니다. 그래서 각도로 색 버킷을 나누고, 같은 색 입자들의 선분을 하나의 Path에 모아 버킷당 딱 한 번씩만 그립니다. 수천 개의 선이 네 번의 draw call로 줄었습니다.

*

트레일(잔상)은 화면을 지우지 않고, 매 프레임 배경색을 6% 투명도로 덮어 만듭니다. 지난 프레임이 서서히 사라지며 비단결 같은 궤적이 남습니다. 지우개 대신 얇은 안개를 쓰는 셈입니다.

보이지 않을 땐 쉰다

스크롤로 캔버스가 화면 밖으로 나가면 IntersectionObserver가 루프를 멈춥니다. 보이지도 않는 그림에 프레임을 태울 이유가 없으니까요. prefers-reduced-motion을 켠 분에게는 애니메이션 대신 한 장의 정지된 흐름을 그려 드립니다. 어지러움은 취향이 아니라 접근성 문제입니다.

결정: 이건 UI가 아니라 그림

이 캔버스만은 사이트 테마를 따르지 않고 늘 어두운 무대를 씁니다. 그림에는 그림에 맞는 조명이 있고, 억지로 라이트 모드에 맞추면 둘 다 어색해집니다. 규칙을 아는 것만큼 언제 깨는지 아는 것도 디자인이라고 생각합니다.
