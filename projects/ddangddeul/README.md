# 땅뜰 — DDANGDDEUL

지도에서 필지를 고르고, 서류를 내고, 에스크로를 거쳐 등기까지 —
토지 거래 전 과정을 브라우저 안에서 재현한 데모.
원본은 FastAPI + SQLite + 바닐라 JS로 만든 토지 정보·조각거래 플랫폼
프로토타입(비공개 레포)이고, 이 페이지는 백엔드와 지도 타일을 목으로
대체한 정적 재현이다. **네트워크 요청 0.**

**Live:** https://lamgul.github.io/projects/ddangddeul/

## 데모가 하는 일

- **인라인 SVG 지적도.** 필지 21개를 폴리곤으로 그린다. 지목/용도지역
  색상 모드 전환, 범례 필터, 드래그 팬, 필지 클릭 → 상세 시트.
- **안전도 진단.** 원본 `sheet.js`의 riskAssess 로직 이식 — 맹지·용도지역·
  시세 부풀리기·지분거래 신호를 점수로 합산해 양호/주의/위험.
- **구매 파이프라인.** 매물 드로어 → 구매 시작 → 가계약 → 본계약 →
  (농지면) 농취증 → 잔금·등기. 서류는 클릭하면 수령 → 검토중 → 승인으로
  목 전환되고, 서류가 다 승인되기 전엔 다음 단계로 못 넘어간다 —
  원본 서버의 409 검사를 클라이언트에서 그대로 재현했다.
- **에스크로.** 가상계좌 입금 → 입금확인 → 등기 완료 시 매도인 지급.
- **상태는 localStorage.** 새로고침해도 거래가 이어지고, 초기화 버튼으로
  처음부터 다시.

## 원본과 다른 점

- 브이월드 WMTS/WFS 타일 → 인라인 SVG 폴리곤
- FastAPI + SQLite → 목 어댑터 (필지·매물·실거래 데이터는
  `backend/db.py`, `sample_data.py` 시드 원본 그대로)
- 로그인·판매 등록 위저드·OCR 검증·서류 반려 흐름은 생략

`ddangddeul.js` 한 파일이 데모의 전부다.
