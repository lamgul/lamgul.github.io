땅뜰.
    필지를 눌러보세요 · 드래그 이동

    토지는 정보가 가장 어두운 자산입니다. 경계·지목·규제·시세가 서로 다른 창구에 흩어져 있고, 계약은 여전히 종이로 오갑니다. 땅뜰은 그걸 하나의 화면으로 모은 실험이었습니다 — 지도에서 필지를 고르면, 등기까지 한 파이프라인으로 흘러가도록.

    실제 시스템: 공공데이터 위의 거래 파이프라인

    원본은 FastAPI + SQLite 백엔드와 프레임워크 없는 바닐라 JS 프론트(모듈 10개)로 이뤄진 프로토타입입니다. 백엔드가 브이월드 연속지적도(WFS)를 약 2km 격자 타일 단위로 받아 SQLite에 적재·캐싱하고, 국토부 RTMS 실거래가와 개별공시지가를 같은 테이블 구조에 얹습니다. 시드 데이터에는 전부 (샘플) 라벨을 붙여 두어, API 키가 등록되면 같은 자리에 실데이터가 들어와 자동으로 교체됩니다. 그 위에 매물 등록(소유자 직접/중개사 위탁), 딜 생성, 에스크로 결제, 서류 수령·검토, 등기 트래킹이 REST API로 이어집니다.

    안전도라는 뱃지의 근거

    기획부동산 피해는 대부분 '몰라서' 생깁니다. 그래서 필지 상세 시트에 위험 점수를 달았습니다. 맹지 여부, 용도지역의 개발 제약, 주변 실거래 대비 가격 부풀리기, 지분거래 비중 같은 신호를 점수로 합산해 양호·주의·위험 세 등급으로 보여줍니다. 원본 sheet.js의 실제 코드 일부입니다.

    if(p.road===false){risk+=30;bad.push("맹지 — 도로 미접으로 건축·개발 제한");}
/* …용도지역·농지·허가구역 가중치… */
if(dev>40){risk+=24;bad.push(`가격이 주변 실거래 평균보다 ${dev}% 높음 — 부풀리기 주의`);}
if(shareR>0.25){risk+=12;bad.push("주변 지분거래 비중 높음 — 기획부동산 주의 지역");}
risk=Math.min(100,risk);
const grade=risk>=55?"위험":risk>=30?"주의":"양호";

    데모의 안전도 진단도 이 로직을 그대로 이식했습니다. 시세 비교에 쓰는 실거래 30건 역시 원본 sample_data.py가 seed(42)로 생성하는 값을 그대로 가져온 것입니다.

    설계 결정: 서류가 문을 여는 진행바

    거래 단계는 버튼을 누른다고 넘어가지 않습니다. 서버가 현재 단계의 필요서류가 전부 수령됐는지를 검사하고, 모자라면 무엇이 빠졌는지 이름을 붙여 409로 거절합니다. 진행바는 UI 장식이 아니라 서류 파이프라인의 상태 그 자체가 됩니다.

    # backend/main.py — advance_deal
missing = [doc["name"] for doc in required if doc["key"] not in have]
if missing and not force:
    raise HTTPException(409, "현재 단계 필요서류 미제출: " + ", ".join(missing))

    데모에서도 같은 규칙을 클라이언트에서 재현했습니다. 서류가 검토중인 채로 [다음 단계]를 누르면, 실제 서버가 돌려주던 것과 같은 문구로 거절당합니다.

    데모가 재현한 것, 생략한 것

    재현한 것: 필지 상세 시트(지목·용도지역·공시지가·주변 실거래·안전도), 매물 드로어, 구매 시작 → 서류 업로드(수령 → 검토중 → 승인) → 에스크로 가상계좌 입금 → 잔금·등기까지 차오르는 진행바. 필지 좌표와 지목·공시지가는 원본 backend/db.py의 시드를 그대로 옮겼고, 거래 상태는 localStorage에 남아 새로고침해도 이어집니다.

    대체·생략한 것: 브이월드 타일 지도는 인라인 SVG 필지 폴리곤으로 바꿨습니다 — 네트워크 요청 없이도 지적도의 핵심(경계·지번·색상 레이어)은 전달되기 때문입니다. 색은 전부 사이트 토큰만 써서 라이트/다크 테마를 그대로 따라갑니다. 로그인·판매 등록 위저드·OCR 검증·서류 반려 흐름은 데모 범위에서 뺐습니다.

    *

    화면의 지번·가격은 전부 샘플입니다. 원본 시스템도 시드 데이터의 모든 표기 문자열에 (샘플) 라벨을 붙여 실데이터와의 혼동을 설계 단계에서 차단했습니다.
