새벽의 커밋Commit Rhythm

커밋 로그는 개발자의 생활 기록이기도 합니다. 언제 뜨겁고 언제 식는지, 무엇으로 갈아타는지. 3년치 기록을 펀치카드·캘린더·언어 분포로 눕혀, "나는 언제 코딩하는가"를 눈으로 봤습니다.

역할데이터 가공 · 시각화
스택D3.js · SVG · Node
코드GitHub ↗

샘플 데이터로 그린 시각화입니다.

3년, 하루하루

GitHub 잔디와 같은 기여도 캘린더. 진할수록 그날 커밋이 많습니다. 띄엄띄엄 비는 칸들은 — 정직하게 말하면 — 번아웃과 휴가입니다. 셀에 마우스를 올리면 날짜별 수가 보입니다.

나는 야행성이다

요일(세로) × 시각(가로). 원이 클수록 그 시간대 커밋이 많습니다. 밤 10시부터 새벽 2시 사이에 뭉텅이가 몰려 있는 게 한눈에 보입니다. 오후 3시쯤의 작은 봉우리는 회의 끝나고 겨우 집중이 붙는 시간대고요.

점점 더 뜨거워지는

월별 커밋을 언어로 쌓았습니다. 시간이 갈수록 막대가 높아지고, 색의 무게중심이 Python에서 TypeScript로 옮겨갑니다. 커리어가 백엔드·데이터에서 제품 전반으로 넓어진 궤적이 색으로 남았습니다.

무엇으로 썼나

3년 전체 언어 비중. 한 가지 스택에 갇히지 않으려고 일부러 여러 언어를 오간 결과이기도 합니다.

정직하게: 이건 가짜 데이터입니다

위 숫자들은 제 실제 git 로그가 아닙니다. 사내 저장소 커밋 시간을 공개 페이지에 그대로 노출하는 건 꺼려졌고, 시각화를 여러 시나리오로 시험하려면 통제된 데이터가 필요했습니다. 그래서 야행성·주중 편향·휴가 공백·언어 이동 같은 패턴을 넣어 결정론적으로 3년치를 생성했습니다. 씨앗값이 고정이라 언제 봐도 같은 그림이 나옵니다.

*

중요한 건 데이터가 아니라 파이프라인입니다. 이 페이지의 렌더링 코드는 { daily, punch, monthly, langs } 모양의 JSON만 받으면 됩니다. 진짜 git log --pretty='%ad|%s' 출력을 같은 모양으로 집계하면, 그대로 내 실제 리듬이 그려집니다.

테마를 따라오는 색

캘린더와 펀치카드는 강조색 위에 fill-opacity만 얹었습니다. 덕분에 라이트에서는 연한 주황부터 진한 주황까지, 다크에서는 어둠 위의 주황 농담으로 — 테마가 바뀌어도 다시 그릴 필요가 없습니다. 색을 절대값으로 박지 않고 배경에 맡긴 결과입니다. (오른쪽 위 토글로 확인해 보세요.)

D3는 자로만

D3는 스케일(scaleSqrt, scaleLinear)과 도형(arc, pie)에만 썼습니다. 나머지 SVG는 손으로 그렸습니다. 라이브러리에 레이아웃까지 통째로 맡기면 편하지만, 좌표가 어디서 오는지 모르게 되는 순간 디버깅이 지옥이 됩니다. 자는 빌려도 그림은 내가 그린다는 감각을 좋아합니다.
