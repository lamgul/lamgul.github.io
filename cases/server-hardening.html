응답 헤더 한 줄이뚫은 구멍

보안 감사에서 지적을 받았습니다. "서버의 버전 정보가 HTTP 응답 헤더에 그대로 노출되고 있어 해킹에 취약합니다." 처음엔 '버전 정보가 노출되는 게 뭐가 문제지?' 싶었습니다.

조금만 생각해 보니 심각한 문제였습니다. 공격자가 서버의 정확한 버전을 알게 되면, 그 버전의 알려진 취약점(CVE)을 찾아 정조준할 수 있습니다. 문이 열려 있다고 광고하는 셈이었습니다.

진단

서버가 스스로를 소개하고 있었다

시스템은 CentOS 7 환경에서 Apache httpd를 쓰고 있었고, 기본 설정에서 Server 헤더에 Apache/2.4.6 (CentOS)처럼 상세한 버전 정보가 노출되고 있었습니다. 이걸 숨기기 위해 ServerTokens 지시자를 조사했습니다.

선택

어디까지 숨길 것인가

ServerTokens에는 여러 단계가 있었습니다.

# ServerTokens 옵션별 Server 헤더 노출 수준
Full     Apache/2.4.6 (CentOS) PHP/7.4.3   # 전부 노출
OS       Apache/2.4.6 (CentOS)             # OS까지
Minor    Apache/2.4                        # 마이너 버전까지
Minimal  Apache                            # 제품명만
Prod     Apache                            # 제품명도 최소화 ← 채택

보안 감사 기준과 실무적 필요성을 함께 고려해 ServerTokens Prod를 적용했습니다. 이렇게 하면 Server 헤더에서 유의미한 정보가 사라집니다.

신중함

간단한 변경일수록 조심스럽게

설정 변경 자체는 한 줄이었지만, 저는 신중하게 접근했습니다. 혹시 이 헤더 정보를 참조하는 모니터링 도구나 로드 밸런서가 있을 수 있기 때문입니다. 개발 환경에서 먼저 테스트하고, 모니터링 시스템에 영향이 없는지 확인한 뒤 운영 환경에 반영했습니다. "간단한 변경"이라는 방심이 장애를 만든다는 걸 알고 있었으니까요.

결과

작은 노출도, 정찰의 시작이다

버전 정보 노출을 없애 감사 지적을 해소하고 CVE 정찰의 표면을 줄였습니다. 화려한 작업은 아니지만, 보안은 종종 이런 작은 노출을 하나씩 닫는 일의 합입니다. 그리고 그 과정에서도 운영 안정성을 함께 지키는 게 실무의 감각이라고 생각합니다.
